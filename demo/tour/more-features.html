Exposing More Features in TinyMCE?

The instance of TinyMCE used in this example offers a limited set of capabilities on the toolbar and no menubar at all. Developers integrating TinyMCE can choose to expose more features for content authors. The following walkthrough demonstrates the configurability and extensibility of TinyMCE.

Click the Basic/Full Editor toggle to reload the editor and see more options. Let’s look at how we do this!

Open /src/xxx/xxx.js in a text editor. This file contains the configuration for the Advanced editor configuration in the micro-blog example application. Here is a step-by-step example:

  1. This example using the table plugin, but you can configure any of plugins you choose. Experiment!

  2. Add the plugin to the plugins string in your TinyMCE configuration.

    The advancedConfig object located in /src/xxx/xxx.js contains a plugins key that references a string containing the names of each of the loaded plugins. This example string contains the value table. Include the name of the plugin(s) in the string for the plugins key for each plugin that you want to load.

  3. Explore the documentation page for the table plugin to add additional configuration options.

    This example defines the options to appear on the floating table toolbar. The table_toolbar configuration option allows you to decide the features you want on that toolbar. Review the documentation for each of TinyMCE’s plugins to ensure the configurations meet your use cases.

Review the advancedConfig object to see all of the additional plugins that are loaded via this configuration.

Choosing and configuring your plugins is an important step in ensuring TinyMCE satisfies the needs of your content authors.
